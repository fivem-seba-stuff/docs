# Seba VIP

**Seba VIP** is a comprehensive VIP membership system for FiveM servers, providing exclusive benefits, tiered memberships, and subscription management for enhanced player experiences.

## Description

Seba VIP transforms your server's monetization strategy with a flexible VIP system that offers multiple membership tiers, exclusive perks, and automated subscription management. Perfect for server owners looking to provide premium experiences while generating sustainable revenue streams.

## Features

### Membership Tiers
- Multiple customizable levels (Bronze, Silver, Gold, Diamond, and custom tiers)
- Flexible benefit system with customizable perks for each tier
- Progressive upgrade paths and tier transition systems
- Time-based memberships with daily, weekly, monthly, and lifetime options

### Exclusive Benefits
- Premium spawn perks including exclusive vehicles, weapons, and items
- Economy bonuses with increased payouts and special discounts
- Priority features including queue priority and support escalation
- Cosmetic rewards with unique skins, animations, and visual effects

### Payment Integration
- Multiple gateway support (PayPal, Stripe, and custom integrations)
- Automated subscription management with recurring payment processing
- Gift system allowing VIP purchases for other players
- Comprehensive refund management with automated processing

## Installation

### Requirements
- FiveM Server (latest version recommended)
- Compatible framework (ESX, QBCore, or Standalone)
- MySQL or MariaDB database
- Web server for payment processing (optional)

### Setup Process

1. Download the latest version from the repository
2. Extract files to your server's resources folder
3. Import the `database.sql` file into your database
4. Configure payment gateway settings in `config.lua`
5. Set up payment integration credentials (if using)
6. Add to server.cfg:
   ```cfg
   ensure seba_vip
   ```
7. Configure VIP tiers and benefits
8. Restart your server

## Server Functions

### VIP Management

#### GiveVIP
```lua
exports.seba_vip:GiveVIP(playerId, tier, duration)
```
Grant VIP status to a player.

**Parameters:**
- `playerId` (number): Player's server ID
- `tier` (string): VIP tier to grant
- `duration` (number): Duration in days

**Example:**
```lua
exports.seba_vip:GiveVIP(1, 'gold', 30) -- Give 30-day Gold VIP
```

#### RemoveVIP
```lua
exports.seba_vip:RemoveVIP(playerId)
```
Remove VIP status from a player.

**Parameters:**
- `playerId` (number): Player's server ID

#### HasVIP
```lua
local hasVIP = exports.seba_vip:HasVIP(playerId)
```
Check if a player has active VIP status.

**Parameters:**
- `playerId` (number): Player's server ID

**Returns:**
- `hasVIP` (boolean): VIP status

#### GetVIPTier
```lua
local tier = exports.seba_vip:GetVIPTier(playerId)
```
Get player's current VIP tier.

**Parameters:**
- `playerId` (number): Player's server ID

**Returns:**
- `tier` (string): Current VIP tier or nil

#### GetVIPExpiry
```lua
local expiry = exports.seba_vip:GetVIPExpiry(playerId)
```
Get VIP expiration timestamp.

**Parameters:**
- `playerId` (number): Player's server ID

**Returns:**
- `expiry` (number): Unix timestamp of expiration

### Benefit System

#### HasBenefit
```lua
local hasBenefit = exports.seba_vip:HasBenefit(playerId, 'spawn_protection')
```
Check if player has specific VIP benefit.

**Parameters:**
- `playerId` (number): Player's server ID
- `benefit` (string): Benefit name to check

**Returns:**
- `hasBenefit` (boolean): Benefit availability

#### ApplyMultiplier
```lua
local amount = exports.seba_vip:ApplyMultiplier(playerId, 'money', baseAmount)
```
Apply VIP multiplier to monetary values.

**Parameters:**
- `playerId` (number): Player's server ID

**Returns:**
- `amount` (number): Amount after VIP multiplier applied

#### GetPlayerBenefits

```lua
local benefits = exports.seba_vip:GetPlayerBenefits(playerId)
```
Get all benefits available to a player.

**Parameters:**

## Usage Examples

### Custom Benefit Implementation

```lua
RegisterServerEvent('myserver:useCustomBenefit')
AddEventHandler('myserver:useCustomBenefit', function()
    local src = source
    
    if exports.seba_vip:HasBenefit(src, 'custom_teleport') then
        TriggerClientEvent('myserver:allowTeleport', src)
    else
        TriggerClientEvent('chat:addMessage', src, {
            color = {255, 0, 0},
            args = {"System", "This feature requires VIP membership!"}
        })
    end
end)
```

### Economy Integration with VIP Bonuses

```lua
RegisterServerEvent('jobs:payPlayer')
AddEventHandler('jobs:payPlayer', function(amount)
    local src = source
    local finalAmount = exports.seba_vip:ApplyMultiplier(src, 'money', amount)
    
    AddMoney(src, finalAmount)
    
    if finalAmount > amount then
        TriggerClientEvent('chat:addMessage', src, {
            color = {0, 255, 0},
            args = {"VIP Bonus", "You received a VIP bonus of $" .. (finalAmount - amount)}
        })
    end
end)
```

### VIP Zone Access Control

```lua
CreateThread(function()
    while true do
        local playerPed = PlayerPedId()
        local coords = GetEntityCoords(playerPed)
        
        if IsPlayerInVIPArea(coords) then
            if not exports.seba_vip:HasVIP(GetPlayerServerId(PlayerId())) then
                SetEntityCoords(playerPed, -1037.5, -2737.6, 20.2)
                ShowNotification('This area is VIP only!')
            end
        end
        
        Wait(5000)
    end
end)
```

### Automated Expiry Management

```lua
CreateThread(function()
    while true do
        local vipPlayers = exports.seba_vip:GetAllVIPPlayers()
        
        for _, player in ipairs(vipPlayers) do
            local expiry = exports.seba_vip:GetVIPExpiry(player.id)
            
            if expiry and expiry < os.time() then
                exports.seba_vip:RemoveVIP(player.id)
                
                if GetPlayerPing(player.id) > 0 then
                    TriggerClientEvent('seba_vip:membershipExpired', player.id)
                end
            end
        end
        
        Wait(300000) -- Check every 5 minutes
    end
end)
```

## Database Schema

### VIP Members Table
```sql
CREATE TABLE `seba_vip_members` (
    `id` int(11) NOT NULL AUTO_INCREMENT,
    `identifier` varchar(50) NOT NULL,
    `tier` varchar(20) NOT NULL,
    `granted_at` timestamp DEFAULT CURRENT_TIMESTAMP,
    `expires_at` timestamp NOT NULL,
    `granted_by` varchar(50),
    `active` tinyint(1) DEFAULT 1,
    PRIMARY KEY (`id`),
    UNIQUE KEY (`identifier`),
    INDEX (`expires_at`, `active`)
);
```

### VIP Transactions Table
```sql
CREATE TABLE `seba_vip_transactions` (
    `id` int(11) NOT NULL AUTO_INCREMENT,
    `identifier` varchar(50) NOT NULL,
    `tier` varchar(20) NOT NULL,
    `amount` decimal(10,2) NOT NULL,
    `currency` varchar(3) DEFAULT 'USD',
    `transaction_id` varchar(100),
    `status` enum('pending','completed','failed','refunded') DEFAULT 'pending',
    `created_at` timestamp DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    INDEX (`identifier`, `status`)
);
```

### VIP Benefits Usage Table
```sql
CREATE TABLE `seba_vip_benefits_log` (
    `id` int(11) NOT NULL AUTO_INCREMENT,
    `identifier` varchar(50) NOT NULL,
    `benefit_type` varchar(50) NOT NULL,
    `benefit_value` text,
    `used_at` timestamp DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    INDEX (`identifier`, `used_at`)
);
```

## Troubleshooting

### Common Issues

#### VIP Status Not Activating
- Verify database connection and ensure all tables exist
- Check that player identifiers are correctly stored in database
- Confirm tier names match exactly (case-sensitive)
- Restart resource after database modifications

#### Payment Processing Issues
- Verify payment gateway configuration and credentials
- Ensure webhook URLs are correctly configured and accessible
- Confirm SSL certificates are valid for production environment
- Test with sandbox/test modes before going live

#### Benefits Not Functioning
- Check benefit configuration in Config.Benefits
- Verify event handlers are properly registered
- Look for conflicts with other server resources
- Enable debug mode for detailed benefit activation logs

#### UI and Menu Problems
- Clear browser cache and reload interface (Ctrl+F5)
- Check browser console for JavaScript errors (F8)
- Verify NUI callback functions are properly configured
- Ensure HTML, CSS, and JS files are intact and unmodified
